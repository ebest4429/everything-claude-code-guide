# CLAUDE.md

## 세션 시작

강제 읽기 순서:
1. 마스터플랜 (`.claude/plans/*-master.md`)
2. 현재 활성 페이지플랜 — `## 작업 항목` 섹션
3. `.claude/WORKSPACE.md` — `## 현재 위치` + `## 남은 과제`

필요 시 추가로 읽는 파일:
- `.claude/CONTEXT.md` — 프로젝트 구조 및 기술 환경
- `.claude/RULES.md` — 작업 원칙

## 페이지플랜 규칙

- 모든 페이지플랜의 현재 작업 섹션 이름: `## 작업 항목` (고정)
- 마스터플랜 상단 `**현재 활성 플랜:** \`파일명\`` 으로 활성 플랜 지정
- `plans/` 직하위 = 활성 중 또는 완료된 페이지플랜만
- `plans/designs/` = 아직 활성화되지 않은 사전설계

**Phase 전환 절차:**
1. `WORKSPACE.md` → `WORKSPACE.{현재플랜stem}.md` 로 이름 변경
2. 새 `WORKSPACE.md` 작성
3. 마스터플랜 활성 플랜 포인터 수정
4. `plans/designs/` 의 사전설계 참조하여 `plans/` 에 새 페이지플랜 Write

## 프로젝트 정체성

**everything-claude-code 가이드 웹사이트**
Claude Code(CLI)의 전체 기능과 사용법을 다루는 가이드 웹사이트.
개요, 설치법, 사용법, 주요 기능, 메뉴별 상세 소개, 스킬/에이전트별 상세 가이드를 포함한다.

마스터플랜: `.claude/plans/everything-claude-code-master.md`

---

## 파일 기록 원칙 — 취지 필수 기재 (모든 파일)

- **결정사항 + 취지를 함께 기록한다**
- 특히 **보류·금지·대기** 항목은 반드시 이유를 함께 기록한다
- 훅·설정파일·구조 변경 시: 어떤 문제가 발생했는지, 왜 이 방법을 선택했는지 기록한다

---

## 파일 내용 보고 원칙 (필수)

- **파일에 실제로 있는 내용만 말한다**
- 파일 외 정보를 언급해야 할 경우 반드시 **(파일 외 정보)** 라고 명시

---

## WORKSPACE.md 운용 원칙 (필수)

- **Edit 도구로만 수정**
- **체크박스 없음**: 서술 방식으로 기록
- **진행 이력**: 최신→과거 순으로 추가

---

## 작업 순서 원칙 (필수)

```
1. 플랜 파일에서 해당 항목 확인
2. 플랜 항목 상태 🔲 → 🔄 표시
3. 구현
4. 플랜 항목 상태 🔄 → ✅ 표시
5. WORKSPACE.md 진행상황 업데이트
6. 커밋
```
