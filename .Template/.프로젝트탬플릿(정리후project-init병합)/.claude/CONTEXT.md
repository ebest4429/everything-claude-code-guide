# CONTEXT.md

## 프로젝트 정체성

**재정계획 스킬 — 지자체 중기지방재정계획 분석 스킬**
Claude CLI 환경에서 `/재정계획` 트리거로 동작하는 챗봇형 스킬.
사용자와 단계별 대화를 통해 지자체별 분석용 파일을 생성한다.

---

## 기술 환경

| 항목 | 내용 |
|------|------|
| 실행 환경 | Claude Code (CLI) |
| 스킬 트리거 | `/재정계획` |
| 스킬 형식 | Claude CLI 커맨드 파일 (`.claude/commands/재정계획.md`) |
| 생성 파일 | `NotebookLM_프롬프트.md`, `Claude_세션1_시작프롬프트.md` |
| 생성 위치 | `재정계획-{지자체명약칭}{순번}/` (순번 자동 부여) |

---

## 프로젝트 범위

| 영역 | 위치 | 설명 |
|------|------|------|
| 스킬 개발 | 이 프로젝트 | `/재정계획` 스킬 제작 |
| 재정 분석 | `재정계획-{지자체명}{순번}/` | 스킬 실행 결과물로 별도 진행 |

---

## 디렉토리 구조

```
midterm-local-fiscalplan/
├── .claude/
│   ├── commands/
│   │   └── 재정계획.md          # 메인 스킬 (작성 대상)
│   ├── hooks/                    # 훅 스크립트
│   ├── plans/
│   │   ├── 재정계획-master.md    # 마스터플랜 (전체 Phase 기준)
│   │   └── {페이지플랜}.md       # 현재 Phase 플랜
│   ├── CLAUDE.md
│   ├── CONTEXT.md
│   ├── RULES.md
│   └── WORKSPACE.md
├── 재정계획-{지자체명약칭}{순번}/  # 스킬 실행 결과물
│   ├── NotebookLM_프롬프트.md
│   └── Claude_세션1_시작프롬프트.md
└── .Source-Files/                 # 참조 전용 문서 (수정 금지)
```

---

## 스킬 동작 흐름

```
사용자: /재정계획 입력
    ↓
Step 1: 지자체명 수집
Step 2: 분석 목적 수집 → 목적 기반 분야 자동 추천
Step 3: 보유 데이터 확인
Step 4: 집중 분야 확정 (추천 조정)
Step 5: 분석 기간 설정
    ↓
파일 2종 생성:
  - NotebookLM_프롬프트.md    (선택 분야 기반 데이터 추출 프롬프트)
  - Claude_세션1_시작프롬프트.md  (목적+분야 기반 분석 시작 프롬프트)
    ↓
재정계획-{지자체명약칭}{순번}/ 폴더에 저장
```
