# RULES.md

## 실행 환경

- OS: Windows 10 Pro
- Shell: bash (Git Bash / Claude Code 기준)
- 실행: Claude Code CLI (`/재정계획` 트리거)

---

## 스킬 작성 원칙

- 스킬 파일 위치: `.claude/commands/재정계획.md`
- 모든 텍스트는 순한글 사용 (한자 금지)
- ZIP 구조 금지 — CLI는 파일 직접 생성
- templates/ 폴더 구조 금지
- 생성 파일 2종만: `NotebookLM_프롬프트.md`, `Claude_세션1_시작프롬프트.md`
- 생성 위치: `재정계획-{지자체명약칭}{순번}/` (기존 폴더 확인 후 다음 번호 자동 부여)
- 행정구역 정보는 AI 일반지식으로 판단 (하드코딩된 데이터베이스 참조 금지)

---

## 절대 금지 (이전 오류 반복 금지)

1. 한글 대신 한자 사용 — 모든 텍스트 순한글 (검증: `析`, `分`, `計`, `劃` 등 한자 문자 grep으로 확인)
2. 불필요한 추가 파일 생성 — 2종만
3. ZIP 구조 사용
4. templates/ 폴더 구조
5. 파일명 오류 (대소문자, 언더바 등)
6. 생성 위치 오류 (위 확정 위치 준수)
7. 프롬프트 내용 엉터리 작성

---

## 플랜 규칙

- 현재 Phase와 플랜은 `WORKSPACE.md`에서 확인
- 전체 Phase 구조는 `재정계획-master.md`에서 확인
- 파일 수정 전 `WORKSPACE.md` + 현재 페이지 플랜 확인 필수
- 완료된 체크박스 즉시 완료 표시로 업데이트
- 플랜에 없는 작업 시작 전 사용자 확인 필수

---

## 전면 변경 시 필수 점검

다음 상황 중 하나라도 해당하면 **플랜 및 지침 파일 전체 점검 후 진행**:

- 플랜에 없던 작업이 추가된 경우
- 스킬 구조를 전면 재작성하는 경우
- 설계 결정이 바뀐 경우
- Phase 완료 조건이 변경되는 경우

**점검 대상:**

| 파일 | 확인 항목 |
|------|----------|
| `WORKSPACE.md` | 현재 상태 최신화 |
| `재정계획-master.md` | 완료 조건, 결정사항 최신화 |
| 현재 Phase 플랜 | 체크박스, 추가 작업 반영 |
| `RULES.md` | 새 원칙이 생긴 경우 추가 |
