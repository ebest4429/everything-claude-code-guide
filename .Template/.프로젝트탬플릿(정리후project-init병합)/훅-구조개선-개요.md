## 현재 구조의 문제점

### 1. protect-workspace.js — 트리거 위치 오류

**현재 동작:**
- `Write` 도구로 `WORKSPACE.md`를 쓰려 할 때 기존 파일 존재 시 차단
- `Edit` 도구는 차단하지 않음 (의도적 — WORKSPACE는 지속 업데이트 필요)

**문제:**
- Phase 전환(새 플랜 생성) 시 WORKSPACE 아카이브 여부를 강제할 수 없음
- 훅이 WORKSPACE.md 쓰기 시점에 걸려 있으나, 실제 Phase 전환 게이트는
  **새 플랜 파일 생성 시점**에 걸어야 함

**개선 방향:**
- `Write` to `.claude/plans/phase*.md` 시점에 훅 추가
- 체크 내용: 이전 WORKSPACE 아카이브됐는가? 마스터플랜 포인터 업데이트됐는가?
- protect-workspace.js는 현행 유지 (2차 방어선으로 충분)

### 2. check-plan.js / check-work-order.js — CODE_DIRS 하드코딩

**현재 상태:**
```javascript
// check-plan.js 18번째 줄
const CODE_DIRS = ['main', 'preload', 'renderer']; // Electron 프로젝트 잔재
```

**문제:**
- 프로젝트 복사 시 이 값을 수정하지 않으면 코드 디렉터리 감지 불가
- 커밋 전 플랜 업데이트 검증이 완전히 무력화됨
- 자동화 불가 — 프로젝트마다 디렉터리 구조가 다름

**개선 방향 (안):**

**안 A — 설정 파일 외부화:**
```json
// .claude/hooks/hooks-config.json
{ "codeDirs": ["popup", "background"] }
```
check-plan.js가 이 파일을 읽어 CODE_DIRS 결정.
새 프로젝트 초기화 시 hooks-config.json만 수정.

**안 B — 자동 탐색:**
프로젝트 루트의 특정 마커(manifest.json, package.json 등)를 보고
코드 디렉터리를 휴리스틱으로 탐색. 오탐 가능성 있음.

**안 C — 현행 유지 + 가이드 보완 (채택):**
CODE_DIRS를 하드코딩 상태로 유지하되, NEW-PROJECT-GUIDE.md 초기화 체크리스트에
"CODE_DIRS 수정" 항목 추가 → 사용자가 협의하여 초기화 시 직접 수정.

> 이 프로젝트에서는 안 C 적용 완료:
> `['main','preload','renderer']` → `['popup','background']` 수정됨
> NEW-PROJECT-GUIDE.md 체크리스트에 항목 추가됨

### 3. session-gate.js — 신규 프로젝트 첫 세션 영구 차단

**현재 동작:**
- SessionStart 훅이 `*-master.md` 탐색 → 활성 플랜 추출 → gate.json 생성
- 신규 프로젝트(구 프로젝트 복사 직후)에서 마스터플랜이 없거나 활성 플랜 경로가
  이전 프로젝트 파일을 가리키면 gate.json의 required 파일들이 존재하지 않음
- Read 도구로 아무리 읽어도 게이트 해제 불가 → 영구 차단

**해결책 (현행):**
```
! rm .claude/.session-gate.json
```
수동으로 게이트 삭제 후 진행. NEW-PROJECT-GUIDE.md Step 3에 명시됨.

**개선 방향:**
- gate.json 생성 시 required 파일 존재 여부 사전 검증
- 파일이 없으면 gate.json을 생성하지 않고 경고만 출력
- 또는 gate.json에 `"bypass": true` 플래그를 두어 사용자가 직접 설정 가능하게
