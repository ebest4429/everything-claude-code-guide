## 분석 방법론

### 사업 생애주기 4단계 분류

```
[1단계] 계획 수립
   ↓ (투자심사 결과에서 포착)
[2단계] 초기 투입 (예타/사전조사비)
   ↓
[3-A단계] 본격 추진 (연속적 예산 증가)
   OR
[3-B단계] 정체기 (예산 미배정/소액 배정)
   ↓
[4단계] 완료 (예산 감소 후 종료)
```

### 우선순위 상승 3가지 유형

| 유형 | 판단 기준 | 의미 |
|------|-----------|------|
| **증가율형** | 전년 대비 50% 이상 증가 | 기존 사업 가속화 |
| **절대액형** | 전년 대비 50억원 이상 증가 | 대규모 투입 결정 |
| **신규형** | 과거 3년간 예산 0 → 신규 배정 | 새로운 정책 방향 |

### 정체 판단 기준

| 상황 | 판단 |
|------|------|
| 3년 연속 예산 미배정 | 사실상 백지화 가능성 |
| 예타 통과 후 2년 이상 연 10억 미만 | 정체 또는 재검토 중 |
| 예산 2년 연속 50% 이상 감소 | 사업 축소/종료 신호 |

## 작업 루틴

### 새 세션 시작 시

1. **workspace.md 읽기**
   - 이전 세션 진행상황 파악
   - 현재 Phase 확인
   - 다음 TODO 확인

2. **context.md 복습**
   - 분석 목적 재확인
   - 데이터 출처 확인

3. **rules.md 준수**
   - 분석 원칙 상기
   - 금지사항 체크

### 분석 진행 중

1. **데이터 검증**
   - NotebookLM 제공 데이터의 일관성 확인
   - 이상치 발견 시 재확인 요청

2. **패턴 인식**
   - 반복되는 투자 패턴 포착
   - 예외적인 변화 주목

3. **가설 수립 및 검증**
   - "X 지역이 우선순위다" → 데이터로 검증
   - 반증 데이터 적극 탐색

### 세션 종료 시

1. **workspace.md 업데이트**
   - 완료한 작업 체크
   - 주요 발견사항 기록
   - 다음 세션 TODO 작성

2. **중간 산출물 저장**
   - 분석 결과를 세션별 .md 파일로 저장
   - 다음 세션에서 참조 가능하도록

2. **정치적 해석 배제**
   - 시장/도지사, 정당, 선거 등과 연결 금지
