Excel 기반 구조를 SQLite 기반 Electron 데스크톱 앱으로 재구축한 로컬 전용 업무 애플리케이션.

## 파일 내용 보고 원칙 (필수)

파일을 읽고 내용을 보고하거나 분석할 때:

- **파일에 실제로 있는 내용만 말한다**
- 파일에 없는 내용을 "누락", "추가로", "참고로" 등의 표현으로 덧붙이는 것 **금지**
- 추론·추정·배경지식을 파일 내용인 것처럼 섞어서 말하는 것 **금지**
- 파일 외 정보를 언급해야 할 경우 반드시 **"(파일 외 정보)"** 라고 명시한 후 사용자 확인을 받는다
- 확인하지 않은 사실을 단정적으로 말하는 것 **금지** — 불확실하면 "확인이 필요합니다"라고 말한다

## WORKSPACE.md 작성 원칙 (필수)

> AI는 이 원칙을 반드시 준수한다. 위반 시 개발 이력이 소실된다.

**기본 규칙**
- **업데이트 방식**: 플랜 진행 중에는 내용 추가/수정만 허용. 전면 덮어쓰기 금지
- **체크박스 없음**: 진행상황은 서술 방식으로 기록. 체크박스는 페이지 플랜에만 존재
- **아카이브 먼저**: 새 WORKSPACE.md 작성 전 반드시 기존 파일을 아카이브로 보존

**초기화 조건**
- 새 페이지 플랜 시작 시 → `archive-workspace` 스킬로 기존 파일 아카이브 후 재작성
- 단, 복수 플랜이 동시 진행 중일 때는 초기화 불가 → 사용자가 수동 판단 후 스킬 호출
- 페이지 플랜 진행 중 추가 구현(임시 플랜, 보완 등)이 발생한 경우 → 사용자가 초기화를 명시적으로 결정하기 전까지 업데이트 방식 유지

**보류 항목 완료 처리**
- 보류 중인 항목이 완료되면 해당 행을 테이블에서 삭제한다
- "구현 완료" 등 텍스트만 바꾸고 행을 남기는 것 금지

**금지**
- 사용자 지시 없이 WORKSPACE.md 전면 재작성 금지
- 이전 Phase 기록을 삭제하거나 덮어쓰기 금지
- 체크박스 형식 사용 금지 (서술 방식으로 기록)

**아카이브 방법**: `.claude/skills/archive-workspace.md` 스킬 참조
