# value-up 기능 보완 마스터 플랜

> Phase 7 동결 후 방향 전환 (2026-03-28)
> Phase 8~10 보류 — 기초 기능 완성도 확보 우선

---

## 이 플랜의 역할

기능 보완 전체를 조망하는 마스터 플랜.
대형 항목(목록탭, 설정탭)은 별도 플랜 파일로 관리하고 이 파일이 인덱스 역할을 한다.
상세 내용은 협의 후 순서대로 채워 나간다.

---

## 플랜 파일 구성도

```
value-up-기능보완.md  ← 지금 이 파일 (마스터/인덱스)
│
├── value-up-목록탭.md        ← 착수 전 생성 (대형 — Phase급)
├── value-up-설정탭.md        ← 착수 전 생성 (대형 — Phase급)
│
└── 구조보완 / 기능재설계 / 결함수정
    → 이 파일에 직접 기록 (소~중형)
```

> 대형 항목 착수 순서: 마스터 플랜 항목 🔲 → 별도 플랜 파일 생성 → 구현 → 완료 후 archive/

## 세션 시작 체크포인트

> 새 세션 또는 컨텍스트 재진입 시 이 순서로 확인한다

1. `WORKSPACE.md` — 현재 위치 + 마지막 작업 확인
2. **이 파일(기능보완 마스터)** — 전체 진행 상황 파악
3. 현재 진행 중인 별도 플랜 파일 (있으면) — 상세 작업 확인
4. 작업 재개

## 돌발 항목 처리 원칙

> 구현 중 계획에 없는 작업이 생겼을 때

| 유형 | 처리 방법 |
|------|---------|
| 소형 버그/결함 발견 | 이 파일 E 섹션에 항목 추가 → 작업 순서 진행 |
| 중형 기능 변경/추가 | 이 파일 해당 섹션에 항목 추가 → 작업 순서 진행 |
| 대형 신규 기능 | 사용자 협의 → 별도 플랜 파일 생성 → 이 파일에 항목 추가 |
| 설계 변경 | 사용자 협의 → 관련 플랜 파일 수정 → 필요 시 마스터플랜도 수정 |
| 다른 Phase 영역 영향 | 반드시 사용자 확인 후 진행 |

**원칙: 계획에 없는 작업은 반드시 플랜 항목 추가 후 진행. 무단 구현 금지.**
