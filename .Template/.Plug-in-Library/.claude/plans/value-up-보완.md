## B3. 아코디언 + 인쇄 연동 설계

### 배경

- 닫힌 아코디언 섹션은 인쇄에서 제외
- 매물종류 선택 시 관련 섹션 자동 활성화

### 설계 방향

```
섹션 상태(열림/닫힘) → 인쇄 시 collapsed 클래스 → no-print 처리
매물종류 → 섹션 매핑 테이블 → 자동 열기/닫기
```

### 구현 범위

- `index.html` `toggleSection()` → 섹션 상태를 class로 관리
- `print.css` — `.section.collapsed { display: none; }`
- 매물종류-섹션 매핑 테이블

### 우선순위 하향 이유

Phase 6(인쇄) 구현 시 함께 처리가 가장 효율적. B7(입력규칙) 및 B5(도로명) 완료 후 진입.

## B6. 프로세스 개선

- 현재: 요청 → 구현 → 체크리스트 → 플랜보완 → 커밋
- 목표: 요청 → **플랜보완 먼저** → 구현 → 체크리스트 → 커밋
- 플랜 체크박스/WORKSPACE 업데이트 없이 다음 단계로 진입하는 패턴 반복

3. 플랜 체크박스 ✅ 업데이트

## B8. 다중 지번 지원

> 소스: `.Source-Files/보완사항.md` — "다중 지번(여러필지)의 경우 지목과 용도지역 여럿존재"

### 협의 결정 (2026-03-27)

| 상황 | 처리 방식 |
|------|---------|
| 한 필지 + 건물 다수 | 비고로 처리 — 별도 구현 불필요 |
| 여러 필지 + 건물 1개 | 토지 섹션 다중지번으로 해결 |
| 토지매물 여러 지번 | 토지 섹션 다중지번으로 해결 |
| 건물 없는 토지 | 도로명주소 없음 — 지번주소만 사용, 다중지번으로 관리 |

### 구현 방향 (확정)

- **토지 섹션에 기본정보의 주소입력 구조(시도→시군구→읍면동→리→본번/부번) 도입**
- 행 추가 방식으로 지번 추가 가능 (임대현황과 유사한 패턴)
- 건물 없는 토지는 도로명주소 필드 불필요 — 토지 다중지번만으로 매물관리
- 신규 테이블: `land_parcels` (property_id, 주소 구조, 지목, 용도지역, 면적 등)

### 미결정 사항

- 기존 토지이용정보(단일 지번) → land_parcels 마이그레이션 방식
- 합계 면적 계산 (다중 필지 합산)
- Phase 범위: Phase 6 완료 후 별도 플랜으로 진행
