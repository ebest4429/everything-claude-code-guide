# 03. 기본 사용법

Claude Code는 상황에 따라 세 가지 방식으로 실행할 수 있습니다. 직접 대화하며 작업할 수도 있고, 스크립트에서 자동으로 호출할 수도 있습니다.

## 실행 모드 요약 (Run Modes)

| 모드             | 명령어                          | 용도                                          |
| :--------------- | :------------------------------ | :-------------------------------------------- |
| **인터랙티브**   | `claude`                        | 대화형 세션. 대부분의 개발 작업에 사용        |
| **원샷 (Print)** | `claude -p "query"`             | 질문 하나에 답변 하나. 스크립팅/자동화에 적합 |
| **파이프**       | `cat file \| claude -p "query"` | 파일 내용이나 명령어 출력을 Claude에게 전달   |

---

## 인터랙티브 모드 (Interactive)

가장 기본적인 사용 방법입니다. 동료 개발자와 대화하듯 터미널에서 자연어로 작업을 지시합니다.

```bash
# 세션 시작
claude

# 초기 프롬프트와 함께 시작
claude "이 프로젝트의 구조를 설명해줘"
```

### 세션 이름 지정

세션에 이름을 부여하면 나중에 쉽게 찾아서 재개할 수 있습니다.

- **시작 시 이름 지정**: `claude --name "auth-refactor"`
- **세션 중 이름 변경 (슬래시 명령어)**: `/rename oauth-migration`
- **대화 이력으로부터 이름 자동 생성**: `/rename` (최소 1개 이상의 메시지 필요)
- **세션 목록에서 선택하여 재개**: `claude --resume` 또는 `/resume`

### 입력 방식

| 방법            | 설명                                                                   |
| :-------------- | :--------------------------------------------------------------------- |
| **일반 입력**   | 텍스트를 입력하고 Enter로 전송. 자연어로 작업을 지시                   |
| **@ 파일 참조** | `@src/index.ts` 형태로 특정 파일/디렉토리를 Claude에게 직접 보여줌     |
| **! Bash 모드** | `! npm test` 형태로 Claude를 거치지 않고 셸 명령어 직접 실행           |
| **/ 명령어**    | `/compact`, `/clear` 등 Claude Code 자체 기능을 제어하는 슬래시 명령어 |

---

## @ 참조 구문 (파일/디렉토리 멘션)

Claude는 필요한 파일을 스스로 찾을 수 있지만, 특정 파일을 확실히 참조하게 하고 싶을 때 `@` 멘션을 사용합니다. 프롬프트에 `@`를 입력하면 파일 경로 자동완성이 활성화됩니다.

```bash
# 파일 참조
@src/components/Button.tsx 이 컴포넌트를 리팩터링해줘

# 디렉토리 참조
@src/api/ 디렉토리의 구조를 설명해줘

# 여러 파일 동시 참조
@package.json @tsconfig.json 빌드 설정을 확인해줘
```

- `@`를 입력하면 파일 경로 자동완성 피커가 표시됩니다.
- `.gitignore` 패턴에 해당하는 파일은 기본적으로 제외됩니다 (`respectGitignore` 설정으로 변경 가능).
- 대규모 모노레포에서 성능이 느리면 `settings.json`의 `fileSuggestion` 설정으로 커스텀 파일 인덱싱 명령어를 지정할 수 있습니다.

## ! Bash 모드 상세

`!` 접두사를 붙이면 Claude를 거치지 않고 bash 명령어를 직접 실행할 수 있습니다.

- `! npm test`, `! git status`, `! ls -la`
- 명령어와 출력이 대화 컨텍스트에 자동 추가됩니다.
- 실시간 진행 상태와 출력을 표시합니다.
- **Ctrl+B**로 백그라운드 전환도 가능합니다.
- **탭 자동완성**: 부분 명령어를 입력하고 `Tab`을 누르면 현재 프로젝트의 이전 `!` 명령어 히스토리를 기반으로 자동완성됩니다.
- **빈 프롬프트에서 Ctrl+U**: Bash 모드(`!`)에서 빈 프롬프트 상태일 때 `Ctrl+U`를 누르면 일반 프롬프트 모드로 돌아갑니다.

---

## 역방향 검색

`Ctrl+R`을 누르면 이전 프롬프트를 대화형으로 검색할 수 있습니다.

- `Ctrl+R`로 검색 시작
- 검색어를 입력하면 일치하는 이전 명령어가 표시됨
- `Ctrl+R`을 다시 누르면 더 오래된 결과로 이동
- `Tab` 또는 `Esc`로 선택 후 편집, `Enter`로 즉시 실행
- `Ctrl+C`로 검색 취소

## 모델 전환

`Alt+P` (macOS에서는 `Option+P`)를 누르면 세션 도중 모델을 변경할 수 있습니다. 현재 작성 중인 프롬프트가 유지된 채로 모델만 바뀝니다.

## Extended Thinking 토글

간단한 코드 수정에는 불필요하지만, 복잡한 버그를 추적하거나 아키텍처를 설계할 때는 Claude가 "생각할 시간"이 필요합니다.
`Alt+T` (macOS에서는 `Option+T`)를 누르면 확장 사고(Extended Thinking) 모드를 활성화/비활성화할 수 있습니다.
_이 단축키를 사용하려면 먼저 `/terminal-setup`을 실행해야 합니다._

| 방법            | 단축키                                             |
| :-------------- | :------------------------------------------------- |
| **이스케이프**  | `\` + `Enter` (모든 터미널)                        |
| **macOS 기본**  | `Option+Enter`                                     |

## 터미널 설정

`/terminal-setup` 명령어를 실행하면 `Shift+Enter`, `Alt+T` 등의 키 바인딩을 설치할 수 있습니다. VS Code, Alacritty, Zed, Warp 터미널을 지원합니다. iTerm2, WezTerm, Ghostty, Kitty는 별도 설정 없이 바로 사용 가능합니다.

## 원샷 모드 (One Shot)

`-p` (또는 `--print`) 플래그를 사용합니다. 대화를 이어가지 않고, 질문 하나에 답변 하나만 받고 끝내는 모드입니다. 셸 스크립트, CI 파이프라인 등 자동화 환경에서 주로 사용합니다.

```bash
# 기본 사용
claude -p "이 프로젝트가 뭘 하는지 설명해줘"

# JSON 출력
claude -p "모든 API 엔드포인트를 나열해줘" --output-format json

# 스트리밍 JSON
claude -p "이 로그 파일을 분석해줘" --output-format stream-json

# 최대 예산 제한 (달러)
claude -p --max-budget-usd 5.00 "query"

# 최대 턴 제한
claude -p --max-turns 3 "query"
```

### 출력 형식

| 형식            | 설명                                                                             |
| :-------------- | :------------------------------------------------------------------------------- |
| **text**        | 일반 텍스트 (기본값). 사람이 읽기 편한 형식                                      |
| **json**        | 모든 처리가 끝난 후 전체 결과를 JSON으로 반환. 후처리 스크립트에서 파싱하기 용이 |
| **stream-json** | 줄 단위로 JSON을 실시간 스트리밍. 긴 작업의 중간 결과를 즉시 확인할 때 유용      |

## 파이프 모드 (Pipe)

Unix 파이프(`|`)를 사용하여 다른 명령어의 출력이나 파일 내용을 Claude에게 전달합니다.

```bash
# 파일 내용 분석
cat README.md | claude -p "이 파일을 요약해줘"

# 에러 로그 분석
cat error.log | claude -p "에러 원인을 분석해줘"

# Git diff 리뷰
git diff | claude -p "이 변경사항을 리뷰해줘"

# 여러 파일 변경사항 리뷰
git diff main --name-only | claude -p "보안 이슈를 검토해줘"
```

## 대화 이어가기 (Resume)

### 가장 최근 대화 이어가기 (-c)

- `claude -c` : 현재 디렉토리의 가장 최근 대화 이어가기
- `claude -c -p "타입 에러를 확인해줘"` : 이어가면서 새 프롬프트 전달 (비대화형)

### 특정 세션 재개 (-r)

- `claude -r "auth-refactor" "이 PR을 마무리해줘"`
- `claude --resume` : 세션 목록에서 선택

### 세션 포크

`claude --resume abc123 --fork-session` : 기존 세션을 복제하여 새 세션 시작

## 체크포인트와 되감기 (Checkpoint)

Claude Code는 파일 편집 상태를 자동으로 추적하여, 문제가 발생했을 때 이전 상태로 빠르게 되돌릴 수 있습니다.

### 자동 체크포인트 추적

- 사용자 프롬프트마다 새로운 체크포인트가 생성됩니다.
- 체크포인트는 세션 간에 유지됩니다.
- 세션과 함께 30일 후 자동 정리됩니다.

### 되감기 메뉴

`Esc`를 두 번 누르거나 `/rewind` 명령어를 사용하여 메뉴를 엽니다.

| 옵션                              | 설명                                   |
| :-------------------------------- | :------------------------------------- |
| **Restore code and conversation** | 코드와 대화 모두 해당 시점으로 되돌림. |
| **Restore conversation**          | 대화만 되돌리고 현재 코드는 유지.      |
| **Restore code**                  | 파일 변경만 되돌리고 대화는 유지.      |
| **Summarize from here**           | 해당 시점 이후의 대화를 요약으로 압축. |
| **Never mind**                    | 변경 없이 목록으로 돌아감              |

### 복원 vs 요약

- 복원은 시간을 되돌리는 것이고, `Summarize from here`는 시간은 그대로 두되 기억을 압축하는 것입니다.
- 요약 시 선택한 메시지 이전은 유지되고, 이후의 모든 메시지는 AI 요약으로 교체됩니다.

### 제한사항

- **Bash 명령어 변경 미추적**: `rm`, `mv`, `cp` 등 bash 명령어로 파일을 수정한 경우 되돌릴 수 없습니다. (Claude의 Edit/Write 도구만 추적)
- **외부 변경 미추적**: 현재 세션 내에서 Claude가 편집한 파일만 추적합니다.
- **버전 관리 대체 불가**: 체크포인트는 실행 취소용이며, 영구 협업에는 Git을 사용해야 합니다.

## Vim 모드

`/vim`으로 활성화하며, 핵심 Vim 키 바인딩을 지원합니다.

### 모드 전환

- `Esc` : NORMAL 모드 진입
- `i`, `I`, `a`, `A`, `o`, `O` : 각종 삽입 모드 진입

### 이동 (NORMAL 모드)

- `h/j/k/l`: 좌/하/상/우
- `w/e/b`: 단어 단위 이동
- `0/$/^`: 줄 처음/끝/첫 비공백
- `gg/G`: 입력 처음/끝
- `f{char}/F{char}`, `t{char}/T{char}`, `;/,` 지원

### 편집 (NORMAL 모드)

- `x`, `dd/D`, `dw/de/db`, `cc/C`, `cw/ce/cb`, `yy/Y`, `p/P`, `>>/<<`, `J`, `.` 지원

### 텍스트 객체

- `iw/aw`, `i"/a"`, `i'/a'`, `i(/a(`, `i[/a[`, `i{/a{`

### 제한사항

- 비주얼 모드(`v`) 및 매크로(`q`) 지원 안 함.
- 커서가 입력의 처음/끝에 있을 때 화살표 키는 히스토리 탐색으로 전환됨.

## 유용한 팁 (Tips)

### 이미지 입력

1. **클립보드 붙여넣기**: macOS(`Cmd+V`), Windows(`Ctrl+V`/`Alt+V`), iTerm2(`Cmd+V`)
2. **파일 경로 지정**: `분석해줘 @screenshots/error.png`
3. **드래그 앤 드롭**: 지원되는 터미널에서 창으로 드래그.

### 프롬프트 제안

회색 제안 프롬프트가 표시되며 `Tab`으로 수락, `Enter`로 즉시 전송 가능합니다. 부모 대화의 캐시를 재사용하여 비용을 최소화합니다.

- 비활성화: `export CLAUDE_CODE_ENABLE_PROMPT_SUGGESTION=false`

### 백그라운드 실행

- "이 테스트를 백그라운드에서 실행해줘"라고 요청.
- 실행 중인 명령어 전환: `Ctrl+B`

## 태스크 리스트 (Task List)

- Claude는 다단계 작업에서 태스크 리스트를 자동 생성합니다.
- `Ctrl+T`로 토글 (최대 10개).
- `/tasks`로 목록 조회 및 관리.
- `CLAUDE_CODE_TASK_LIST_ID` 환경변수로 세션 간 태스크 공유 가능.

---

## PR 리뷰 상태

열린 PR이 있는 브랜치에서 작업 시 풋바에 PR 링크(예: "PR #446")가 표시됩니다.

- **초록**: 승인됨 (approved)
- **노랑**: 리뷰 대기 중 (pending review)
- **빨강**: 변경 요청됨 (changes requested)
- **회색**: 초안 (draft)
- **보라**: 병합됨 (merged)
- `Cmd/Ctrl + 클릭`으로 브라우저에서 열기 가능.
- 60초마다 갱신되며, `gh` CLI 설치 및 `gh auth login` 인증이 필요합니다.
