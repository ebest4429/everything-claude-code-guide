# Claude Code 플러그인 제어 가이드

## 구조 개요

| 파일 | 범위 | 용도 |
|------|------|------|
| `~/.claude/settings.json` | 전역 | 플러그인 설치·활성화 기본값 |
| `.claude/settings.json` | 프로젝트 공유 | 팀 공통 설정 |
| `.claude/settings.local.json` | 프로젝트 로컬 | 개인 오버라이드 (git 미포함) |

---

## 1. OMC (oh-my-claudecode) 프로젝트별 비활성화

OMC는 user scope 전역 설치 전용. 프로젝트별 설정 공식 지원 없음.
Kill switch 환경변수로 일시 중단 가능.

### 비활성화

`.claude/settings.local.json`에 추가:

```json
{
  "env": {
    "DISABLE_OMC": "1"
  }
}
```

### 특정 훅만 건너뛰기

```json
{
  "env": {
    "OMC_SKIP_HOOKS": "SessionStart,PreToolUse"
  }
}
```

### 재활성화

`"DISABLE_OMC"` 항목 삭제 또는 값을 `"0"` 으로 변경.

## 2. bkit / bkit-starter 비활성화

### 결론 (2026-04-02 검증 완료)

프로젝트 레벨 `enabledPlugins: false`는 **전역 설치 플러그인에 적용되지 않음**.
유일한 해결책은 **전역 설정에서 직접 비활성화**.

### bkit-starter 전역 비활성화 (권장)

`~/.claude/settings.json`:
```json
"enabledPlugins": {
  "bkit-starter@bkit-marketplace": false
}
```

**bkit-starter는 초보자 온보딩 전용** (`/first-claude-code`, `/learn-claude-code` 등).
bkit 본체(PDCA, 에이전트팀 등)는 전혀 영향 없음.

### 충돌 원인 확인

`~/.claude/plugins/cache/bkit-marketplace/bkit-starter/1.0.0/hooks/session-start.sh`
→ SessionStart 시 "bkit - Starter 세션 시작" 메시지를 강제 주입
→ 프로젝트 고유 SessionStart 훅과 충돌

### 프로젝트 레벨 비활성화 (제한적 — 참고용)

버그 존재 (GitHub Issue #27247, #25086):
`settings.local.json`에만 있으면 무시됨 → `settings.json`에도 키 필요.
그러나 **전역 설치 플러그인은 프로젝트 레벨로 비활성화 불가** (근본 한계).

### 재활성화

`~/.claude/settings.json`에서 `false` → `true` 변경.

---

## 세션 지속성

`.claude/settings.local.json`과 `.claude/settings.json`은 **파일 기반**이므로
세션이 종료되어도 설정이 유지됨. 재활성화하려면 직접 파일을 수정해야 함.
